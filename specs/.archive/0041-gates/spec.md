# Spec 0041 — gates

Measurement-hygiene gates — exclusive endpoint (start + per-block, recorded), driver-scoped bounded residency, live tests opt-in.

## Why

0040's first pilot run was invalidated (outcome-blind, an hour lost, a full pilot's cells discarded) because a plain `pytest -q` fired LIVE integration tests at the SAME Ollama the pilot was using. Ollama serves a model serially, so suite requests queued against pilot requests → queued turns ate the 240s wall-clock (recorded as honest-empty) or pushed calls past the 300s HTTP timeout (recorded as model-unreachable); and those tests TOUCHED other model tags, which under infinite keep-alive PINNED them resident (8b+4b = 14.3 GB) and memory-squeezed every 14b cell. Queueing and pinning weren't competing causes — one caused the other.

The damage class is the worst kind: contended-endpoint cells are INDISTINGUISHABLE from honest empties and real typed timeouts without an external control (0036's 14b profile is what caught it). This spec makes that contamination structurally impossible before the enlargement/bake-off spend hours of live compute on it.

Ref: 0040 (the invalidated run-1, `_evict_other_models`, findings.md), 0031–0038 (the trustworthy-verifier arc: a run must PROVE its conditions, not assume them), the degrade-masks-outcome convention, 0037 (sent ≠ honored: the `/v1` compat layer silently drops fields), 0038 (probe-first: no wiring before a committed typed outcome).

### Invariants

- **Refuse, don't bypass — the gate is not a lever.** The live driver REFUSES TO START if the Ollama endpoint is not exclusively available, with a stable typed stop (`exclusive-endpoint-contended`, non-zero exit, zero cells run) plus a human-readable message. No warning-and-proceed mode, no flag: the no-bypass property is pinned mechanically (signature introspection, the 0039 `run_ab_paired` precedent), and the only sanctioned unblock is changing the environment. A run that starts against a contended endpoint produces invalid cells that look exactly like real results — the same reason the power gate has no force parameter and the verifier fails a run it can't prove.
- **Exclusivity is RECORDED at its actual strength, never overclaimed.** The artifact carries EVERY exclusivity check (result + timestamp), not just the first, plus an `exclusivity_check_kind` label naming what was actually checked (`start-plus-per-block` for this spec). A start-gate check proves exclusive-at-check-time only; the per-block re-checks bound the window a contamination can hide in to one block, and the artifact claims exactly that — block-granularity auditability, not continuous run-duration proof (the epistemic-labeling discipline of 0039/0040: the claim never exceeds the mechanism). The check's evidence source is `/api/ps`, which exposes RESIDENT MODELS (tag + `expires_at`), not queued or in-flight requests — so the `exclusivity_check_kind` semantics name TWO residuals the check cannot see: (a) the intra-block window (a foreign tag that loads and fully unloads between two checks), and (b) SAME-TAG contention (a concurrent client sending requests to a tag inside the frozen model set — no foreign tag ever loads, every check passes; the 0040 incident was catchable only because the suite touched OTHER tags). Residual (b) is carried by the deselect default (the known contaminator was the test suite) and the single-operator context, not by this check — stated in the artifact, never implied covered. A past run's validity is auditable from its artifact — same posture as model-identity, tool-names, reasoning-tokens, serving_transport.
- **Sent ≠ honored (the 0037 lesson applied to this spec's own knob).** Any residency-bounding mechanism is proven by observed server behavior (`/api/ps` `expires_at`), never by asserting a field on an outbound request. No wiring before a committed typed probe outcome (the 0038 discipline).
- **Safe-by-default test invocation, with an enforced live consumer.** Live integration tests are OPT-IN, not opt-out: marker-based deselect in the committed config so `pytest -q` cannot reach the accident. And the opt-in path has a named EXECUTABLE consumer — the operator drivers assert/exercise it — so the live suite cannot silently rot (a gate that hides the live tests forever is its own failure).
- **Hygiene only — the SUT request body is UNTOUCHED.** This spec changes the eval DRIVERS, driver preflight, test config, and driver-side residency control. It does NOT touch the explorer, tools, gateway semantics, verifier facts, or any capability path — specifically, the production `ModelGateway` outbound request body is byte-unchanged, so the pinned 0034/0038 invariant `explorer_think=None ⇒ params == {max_tokens: 2048}` SURVIVES VERBATIM (no supersession), regression-asserted. No eval numbers change.

## What

- **EXCLUSIVE-ENDPOINT GATE (start + per-block, decided — resolves prior OQ1):** the live driver queries Ollama `/api/ps` for resident models, BEFORE the run and again BEFORE EACH BLOCK (`/api/ps` exposes residents + `expires_at`, NOT queued/in-flight requests — the gate checks and claims resident-set exclusivity, per the two named residuals in the invariant above). The endpoint query is routed behind the same loopback assertion as all gateway egress (`gateway.assert_local` first — the 0019 preflight rule; `/api/ps` is the same loopback-gated class as `/api/tags`). **Foreign resident is a pinned predicate: a resident tag NOT in the frozen run config's model set** (the driver's own block-by-block loads never self-trigger). Start check fails → typed stop `exclusive-endpoint-contended`, non-zero exit, zero cells. A per-block re-check fails → the driver stops BEFORE that block, records the contamination boundary (the failed check's timestamp), and types every cell recorded since the last clean check as suspect — outcome-blind, at boundary granularity, never by looking at outcomes. Worked example: clean check before block N−1, failed check before block N → block N never runs; block N−1's cells (all cells since the last clean check) are typed suspect; blocks 1..N−2 remain valid under their own recorded clean checks. **Suspect-cell disposition (a third `_cell_needs_run` branch, distinct from clean and typed-degrade):** suspect cells are invalidated and archived beside the run (the 0040 contaminated-run posture) and become re-runnable ONLY after a subsequent clean gate check — clean cells still NEVER re-run, typed degrades still get exactly one bounded re-run. A full lease/lock protocol is considered and DECLINED with recorded rationale (single-operator dev host; the per-block re-check bounds the foreign-tag exposure window to one block; the two unseeable residuals are named in the artifact rather than mechanized away).
- **EXCLUSIVITY PROOF IN THE RUN-LEVEL ARTIFACT (named target, version-gated):** the proof rides the RUN-LEVEL driver ledgers (the `PoolPilotLedger` family and successor bake-off/enlargement ledgers) — NOT the per-case verifier artifact (exclusivity is a run-level fact; keeping it off the per-case artifact avoids the 0033/0034/0038 dual-seam class entirely). Schema bump is ADDITIVE and VERSION-GATED per the 0026/0036 pattern: the new version REQUIRES the exclusivity record (all checks + timestamps + `exclusivity_check_kind` + the frozen model set the foreign-predicate was evaluated against); legacy versions (`0040/pilot/1` etc.) keep validating unchanged. Under the new version, an artifact lacking the proof is rejected.
- **DRIVER-SCOPED BOUNDED RESIDENCY, PROBE-FIRST (decided seam — the production gateway is not the seam):** the residency bound is enforced FROM THE DRIVER via the native API (the seam where `_evict_other_models` already lives), never by adding fields to the SUT's `/v1` request body. Mechanism decided by a committed, schema-versioned, spec-local typed-outcome probe: issue a bounded-`keep_alive` native touch against a model pinned at `keep_alive=-1`, then read `/api/ps` `expires_at` — outcomes `{touch-rebounds, touch-ignored}`. Branch touch-rebounds → the driver bounds residency with a native touch per block (primary), `_evict_other_models` retained as defense-in-depth. Branch touch-ignored → eviction remains the only residency control (recorded), and the exclusivity gate + deselect default carry the pinning risk alone. Wiring is conditional on the recorded outcome (loud-fail tripwire on wiring↔evidence drift, the 0038 posture). The residency bound value is pinned in the frozen config, tuned ≥ one block's cadence so bounding never induces intra-block reload churn. **Named residual gaps (not covered by this mechanism, by design):** Deep's `RlmBackend` owns its own LM client, and stray non-driver traffic (e.g. live tests) — both are exactly what the deselect default and the exclusivity gate exist to keep off the endpoint during a measurement; a regression guard asserts Deep and explorer outbound bodies acquire NO new field.
- **DESELECT DEFAULT WITH AN ENFORCED CONSUMER:** marker-based config (pytest.ini/pyproject) so the default invocation excludes live integration tests; running them requires an explicit opt-in flag. The opt-in command is documented AND exercised by a named executable consumer (the operator drivers' preflight asserts the opt-in path works / the live suite is runnable via `require_live_stack`), so opt-in cannot decay into never-run.

## Acceptance criteria

([unit]=fakes; [integration]=live, skip-not-fail)

1. [unit] Exclusive-endpoint preflight: a fixture whose `/api/ps` shows a foreign resident (per the pinned predicate: tag not in the frozen run config's model set) → driver REFUSES with typed stop `exclusive-endpoint-contended`, non-zero exit, zero cells executed. No bypass/force parameter exists — pinned by signature introspection (0039 precedent). The driver's own configured models never trigger the predicate (asserted). The recorded `exclusivity_check_kind` semantics name the two unseeable residuals (intra-block window; same-tag contention) — asserted present in the artifact.
2. [unit] Mid-run contention (the race case): endpoint clean at start, foreign resident appears before block N's re-check → driver stops BEFORE block N; the contamination boundary (failed check + timestamp) is recorded; cells since the last clean check (block N−1's) are typed suspect; blocks 1..N−2 remain valid under their own recorded clean checks — outcome-blind (asserted on a fake endpoint that flips between blocks). Suspect cells resume per the third `_cell_needs_run` branch: invalidated-and-archived, re-runnable only after a clean gate check; clean cells never re-run (asserted).
3. [unit] The run-level ledger schema bump is additive and version-gated: the new version REQUIRES the full exclusivity record (every check + timestamp, `exclusivity_check_kind`, the frozen model set used by the predicate) and the validator rejects a new-version artifact lacking it; legacy versions (`0040/pilot/1` et al.) still validate unchanged (both directions asserted).
4. [unit] Residency-probe machinery: the probe emits a schema-versioned typed outcome in `{touch-rebounds, touch-ignored}`; honoring is judged ONLY from observed `/api/ps` `expires_at` movement, never from the sent request; a loud-FAIL tripwire pins wiring↔committed-outcome consistency (drift fails, never skips). `_evict_other_models` retained as defense-in-depth (regression-pinned).
5. [unit] SUT-boundary regression guards: explorer outbound params for `explorer_think=None` remain byte-identical `{max_tokens: 2048}` (the 0034/0038 pin, unchanged); Deep outbound carries no new field; the residency mechanism lives only in the driver/native-API seam (asserted).
6. [unit/config] Default test invocation deselects live integration tests (marker-based); a test asserts the default selection contains zero live-marked tests. The opt-in flag works AND has a named executable consumer: the operator driver preflight runs `pytest --collect-only` with the opt-in selection and asserts it collects a non-zero count of live-marked tests (and the default selection collects zero) — a mechanical assertion, not documentation-only.
7. [integration] Live probe run: the committed residency-probe driver executes against the real endpoint and records the typed outcome artifact (skip-not-fail without the live stack, via `require_live_stack`).
8. [integration] Live gate run: a driver run on an exclusive endpoint PASSES the gate and its ledger carries the full recorded proof; a run attempted against a deliberately-contended endpoint STOPS with the typed stop (operator-verifiable, skip-not-fail). On the touch-rebounds branch, reload-churn attribution is operationalized, not judged at close: the per-case typed-degrade set is compared against the 0040 clean-run degrade profile on the shared pinned cases, and the driver records an observed-reload marker (an `expires_at` reset since the previous cell) alongside any degrading cell — a NEW degrade counts as churn-attributable only with the marker present.
9. [doc] The 0040 run-1 contamination recorded as the motivating incident; conventions updated: exclusive-endpoint is a hard gate (start + per-block, recorded at its actual strength, two named unseeable residuals); live tests are opt-in with a named executable consumer; residency bounds are driver-scoped and probe-proven (sent ≠ honored). The update explicitly AMENDS the 0040 run-granularity invalidation convention: run-granularity invalidation when no per-check records exist, boundary-granularity when they do — the same outcome-blind criterion, a better-bounded window, never per-suspicious-cell.

## Out of scope

- Pool enlargement; the bake-off; the 0039 thinking A/B.
- The clean-run heavy-repo degrades (5 persistent model-unreachable on heavy repos SURVIVED the clean isolated re-run — a REAL, separate constraint, diagnosed in its own spec, NOT explained by this contamination).
- Any change to the production `ModelGateway` request body, the explorer, tools, gateway semantics, or verifier facts; symbols-tool adoption; any capability or explorer change.
- A lease/lock exclusivity protocol (declined with recorded rationale; see What — the per-block re-check is the decided strength for a single-operator dev host).
- Changing the dev host's server-side `keep_alive=-1` policy itself (operator environment, not repo code; the driver-side bound works against it and the probe measures against it).
