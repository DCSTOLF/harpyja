# Spec 0040 — pool

## Why

Before pool enlargement, do a three-model preflight + pilot + per-pair bake-off power pre-check.

The bake-off expands from 2 models to 3 — `qwen3:14b`, `qwen3:8b`, `qwen3.5:4b` (three sizes, two generations) — because same-model contrast (0039 A/B) was under-powered, and three models give THREE pairwise contrasts (14b-8b, 14b-4b, 8b-4b), generating far more discordance to clear the min-discordant=8 floor. The 8b-vs-4b cut (newer-smaller vs older-larger) is the comparison Harpyja's ~16B memory box most needs answered.

But two of the three models (`qwen3:8b`, `qwen3.5:4b`) have NEVER run through this harness: no coherence proof, no tool-calling/serving proof, no conceptual-stratum locate-count. So before any power-precheck or bake-off, this spec (1) preflights all three models for coherence + tool-calling + think-control, (2) pilots all three on a pinned pilot subset meeting the frozen coverage minimum, (3) computes PER-PAIR power ceilings vs the floor. It decides, per pair, whether the bake-off runs on the current 19-case set or needs pool enlargement — cheaply, before ~hours of live compute.

Ref: 0039 (power pre-check arithmetic + typed STOP + PREREGISTERED config), 0036 (reachability-tagged set, 15 conceptual / 4 lexical, floor 8, pilot 14b 5/10 & 4b-instruct 0/10), the 16B-gibberish lesson (coherence preflight before capability), 0037/0038 (serving/think-control is model+version specific — `/v1` drops `think`, honors `reasoning_effort`), 0031–0035 (trustworthy verifier).

### Invariants

- **Preflight before capability (the 16B-gibberish lesson):** each model must PASS coherence + tool-calling + think-control preflight BEFORE it produces any capability number. A model that emits gibberish, malformed tool_calls, or ignores the think param produces empties that fake "can't localize" — the exact trap the 16B gibberish nearly recorded. ALL THREE models run the SAME typed preflight under the current harness build — `qwen3:14b`'s 0039 history is prior evidence, not a preflight pass; if 14b itself fails re-confirmation, all three pairs are voided with the typed disposition below (never infra ambiguity or partial artifacts).
- **Serving is model+version specific (the 0037/0038 lesson):** confirm each model's tool-calling AND think-control on THIS Ollama `/v1`. `reasoning_effort` honored on `/v1` for 14b (0038) may differ for a newer generation (qwen3.5) — probe, don't assume; pin per model. A model whose think-control is a no-op can still bake off (default-on), but record it (it can't be a thinking-arm later).
- **Typed preflight outcomes, enum + precedence committed BEFORE the probe (the 0023/0037 total-answer-space discipline):** the per-model preflight returns exactly one value of a committed enum — `PREFLIGHT_PASS` / `UNSERVABLE` / `COHERENCE_FAIL` / `TOOL_CALL_MALFORMED` / `THINK_CONTROL_NOOP` — enumerated in code before any probe fires. "Exactly one value" requires a committed tie-break, not implementer choice: precedence `UNSERVABLE > COHERENCE_FAIL > TOOL_CALL_MALFORMED > THINK_CONTROL_NOOP > PREFLIGHT_PASS` (cheapest/most-fundamental failure first; a gibberish model that also emits malformed tool_calls types `COHERENCE_FAIL`). An INDETERMINATE think-control probe (a param whose effect cannot be adjudicated under the tiny-cap discriminator) maps to `THINK_CONTROL_NOOP` — conservative (barred from a thinking-arm), stated here so the probe cannot stall outside the committed answer space. The enum is DELIBERATELY ASYMMETRIC and that asymmetry is load-bearing (state it in the artifact so it is never "fixed" into symmetry): `UNSERVABLE` / `COHERENCE_FAIL` / `TOOL_CALL_MALFORMED` are EXCLUDING (the model produces no capability number); `THINK_CONTROL_NOOP` is RECORDED-NON-EXCLUDING (the model still bakes off default-on; it is only barred from a future thinking-arm).
- **Frozen config before live data (the 0036 re-registration rule):** the 0039 freeze pins one model and two think-arms — it cannot govern a 3-model / 3-pair pre-check. A NEW `PREREGISTERED_POOL_CONFIG_0040` (frozen dataclass + `config_hash`) is committed BEFORE any live pilot call: the three model tags, the three named pairs, the conceptual floor copied VERBATIM from 0039 (`MIN_DISCORDANT_PAIRS=8`), the multiplicity stance (below), the pinned pilot case IDs, the coverage minimum WITH its derivation (below), the preflight enum + precedence, the pair-verdict predicate order (below), the two projection quantities with their epistemic-kind labels (below), the pilot arm parity pin, and the staging order fallback (OQ3). Every per-pair verdict is a total pure function over this frozen object.
- **Multiplicity decided NOW, outcome-blind, frozen:** the floor of 8 was derived for a single McNemar contrast. This spec freezes the stance **per-pair α, uncorrected** — each pair answers a distinct standalone decision question ("does A beat B for the ~16B box"), not a family-wise "does any pair differ"; the 8b-vs-4b cut is independently decision-relevant regardless of the other two pairs. The stance and this rationale are recorded in `PREREGISTERED_POOL_CONFIG_0040` before any pilot data exists; deciding it after seeing which pairs clear 8 is exactly the post-hoc steering the freeze forbids.
- **TWO pinned per-pair quantities, each labeled by its epistemic kind (the bound-is-not-an-estimate discipline, both directions):** from the retained per-case pairs `(case_id, a_bucket, b_bucket)` on the conceptual stratum:
  1. **The CEILING (a true upper bound; `projection_kind="upper-bound-feasibility"`): the extrapolated per-case UNION-located count** — pilot cases where ≥1 of the two models locates, scaled to the 15-case stratum. This bounds max possible signal discordance because `is_signal_discordant` requires ≥1 located arm by its own definition (one-oracle reuse justifying the bound). It gates `PAIR_UNDER_POWERED` — the stop-quality claim rests ONLY on this bound.
  2. **The OBSERVED signal-discordance (a labeled POINT ESTIMATE, `estimate_kind="point-estimate"`): pilot pairs through `is_signal_discordant`, extrapolated to the stratum.** It drives the `PAIR_MODELS_TOO_CLOSE` vs `PAIR_FEASIBLE` split — a REPORTABLE closeness finding, never the unimpeachable stop.
  Extrapolating observed discordance and calling it a bound would be an epistemic mislabel (a false UNDER_POWERED from sampling noise); a literal max-possible bound over unpiloted cases is vacuous (unobserved mass alone clears the floor). The split resolves both failure modes. NEVER compute either quantity from marginal locate-counts: 6/7 vs 5/7 is identical whether the located sets fully overlap (TOO_CLOSE) or are nearly disjoint (highly discordant); union-located and discordance are per-case properties marginals cannot recover. (0039 used counts because within-model think-flips cannot be paired cross-model; 0040 HAS direct cross-model per-case pairs — counts-only here would be an evidence-quality regression, not a reuse.)
- **Pilot coverage minimum, DERIVED not guessed:** `MIN_PILOT_CONCEPTUAL_COVERAGE = 8`, derived from the consuming test's own arithmetic: the unpiloted conceptual remainder `15 − c` must be strictly less than the floor 8 (i.e. `c ≥ 8`), else a verdict could rest on majority-unobserved mass — the exact vacuity boundary (0 observed + 8 unpiloted ≥ floor) that makes a literal bound worthless. Consequence, stated now: the 0036 first-10 subset covers only 7 conceptual cases — BELOW the minimum — so the pinned pilot set extends it by ≥1 conceptual case from the 19-case set (pinned case IDs in the frozen config). A pair whose per-case pairs cover fewer than 8 conceptual cases types `INSUFFICIENT_PILOT_EVIDENCE`, from the frozen predicate, never a read-time judgment.
- **Total per-pair verdict with committed, non-overlapping predicate ORDER:** the per-pair enum is `PAIR_NOT_EVALUATED_MODEL_EXCLUDED` / `INSUFFICIENT_PILOT_EVIDENCE` / `PAIR_UNDER_POWERED` / `PAIR_MODELS_TOO_CLOSE` / `PAIR_FEASIBLE`, evaluated in exactly that frozen order:
  1. `PAIR_NOT_EVALUATED_MODEL_EXCLUDED` — a member model was EXCLUDED at preflight (absence is never a disposition; a 14b re-confirmation failure voids all three pairs with this typed value).
  2. `INSUFFICIENT_PILOT_EVIDENCE` — conceptual coverage `c < 8` for this pair.
  3. `PAIR_UNDER_POWERED` — ceiling (union-located bound) `< 8`: the floor is unreachable even in the best case.
  4. `PAIR_MODELS_TOO_CLOSE` — ceiling `≥ 8` but extrapolated observed discordance `< 8`: clearing is possible only if the models disagree far more on unpiloted cases than they did on piloted ones (estimate-labeled, reportable).
  5. `PAIR_FEASIBLE` — ceiling `≥ 8` and extrapolated observed discordance `≥ 8`.
  A pair under-powered because the two models perform IDENTICALLY (few disagreements) is thus a distinct, reportable finding from under-powered-because-few-cases. Upper-bound honesty stands: `PAIR_FEASIBLE` means "possible to clear the floor," proven only by the bake-off's own run. No forced proceed — the gate carries no force/bypass parameter.
- **Pilot arm parity:** all three models run at the SAME think setting — `explorer_think=None` (default-on) — pinned in the frozen config, so the locate-counts are comparable power inputs measuring model capability, not a thinking confound. A model whose think-control is a no-op runs identically under this pin (default-on); a model that CANNOT run default-on is a preflight exclusion, not a silently different arm.
- **No capability claim here:** this spec preflights, pilots, and computes feasibility. It does NOT rank the models or run the full bake-off — the pilot locate-counts are power inputs, not the ranking. The ranking is the bake-off's own powered run.

## What

- **FROZEN CONFIG FIRST:** commit `PREREGISTERED_POOL_CONFIG_0040` (frozen + hashed, contents per the invariant above) before any live preflight/pilot call fires.
- **PREFLIGHT** (ALL THREE models, committed probe, loud): coherence (known-localizable case → sane output), clean `tool_calls` on `/v1`, and think-control mechanism (`reasoning_effort` or its qwen3.5 equivalent — probe it). Result is exactly one value of the committed preflight enum under the committed precedence, drift-pinned. An EXCLUDING outcome (`UNSERVABLE` / `COHERENCE_FAIL` / `TOOL_CALL_MALFORMED`) removes the model with the typed reason recorded — never silently carried — and types every pair containing it `PAIR_NOT_EVALUATED_MODEL_EXCLUDED`; `THINK_CONTROL_NOOP` is recorded and the model proceeds (default-on). `qwen3:14b` runs the same enum (re-confirmation, not assumption).
- **PILOT:** run all preflight-passing models on the pinned pilot set (0036 first-10 subset + enough additional conceptual cases to meet the frozen coverage minimum of 8) through the trustworthy verifier at `explorer_think=None`; per model+case durable artifact (bucket, tools incl. symbols-adoption, reasoning tokens, submitted/surviving, model identity, `serving_transport`). Record each model's conceptual-stratum locate-count AND retain the per-case buckets for pairing. The committed operator driver STOP-AND-WARNs on infra (unservable tag, no tool-calling, Ollama down) and maintains a RESUMABLE per model+case ledger (exit 3 while work remains) — the ~100+ min pilot outlasts one invocation, same posture as 0039's ledger. Pytest integration wrappers skip-not-fail on an absent live stack; the driver never silently skips.
- **PER-PAIR PRE-CHECK:** for each pair (14b-8b, 14b-4b, 8b-4b), build the retained per-case pairs `(case_id, a_bucket, b_bucket)` on the conceptual stratum and compute the TWO pinned quantities — the union-located CEILING (upper bound) and the observed signal-discordance via `is_signal_discordant` (labeled point estimate) — then emit the typed per-pair verdict through the frozen predicate order, plus the overall fork. Floor comparison reuses 0039's predicate arithmetic (not re-derived); `INSUFFICIENT_PILOT_EVIDENCE` fires from the frozen derived coverage minimum, not a read-time judgment.

## Acceptance criteria

([unit] = fakes; [integration] = live pilot/preflight — pytest wrappers skip-not-fail on absent stack, committed operator drivers STOP-AND-WARN)

1. [unit] `PREREGISTERED_POOL_CONFIG_0040` frozen dataclass + `config_hash` committed and pinned BEFORE live evidence: three model tags, three named pairs, floor 8 verbatim, per-pair-α multiplicity stance + rationale, pinned pilot case IDs, `MIN_PILOT_CONCEPTUAL_COVERAGE=8` WITH its derivation (`15 − c < 8`), preflight enum + precedence, pair-verdict predicate order, both projection quantities with epistemic-kind labels (`projection_kind="upper-bound-feasibility"` for the ceiling, `estimate_kind="point-estimate"` for observed discordance), arm-parity pin (`explorer_think=None`), staging order. Verdicts are total pure functions over it.
2. [unit] Preflight outcome enum + precedence committed before any probe: exactly the five values; tie-break asserted on a fixture failing BOTH coherence and tool-calling (types `COHERENCE_FAIL`, not implementer choice); indeterminate think-probe maps to `THINK_CONTROL_NOOP`; asymmetry asserted by test — `THINK_CONTROL_NOOP` is non-excluding (model proceeds to pilot), `UNSERVABLE`/`COHERENCE_FAIL`/`TOOL_CALL_MALFORMED` exclude with the typed reason recorded.
3. [integration] ALL THREE models PREFLIGHT run live under the same enum (14b re-confirmed, not assumed from 0039): coherence + clean `/v1` tool_calls + think-control probed; typed per-model result committed + drift-pinned; a failing model EXCLUDED with recorded typed reason (not silently included).
4. [integration] Three-model PILOT on the pinned pilot set at `explorer_think=None`: per model+case verifier-clean artifacts with per-case buckets retained; each model's conceptual locate-count recorded; driver STOP-AND-WARNs on infra and resumes from its ledger.
5. [unit] BOTH per-pair quantities computed for all 3 pairs FROM per-case pairs (identity-asserted one-oracle reuse of `is_signal_discordant`; 0039 floor-comparison arithmetic reused, not re-derived): the union-located ceiling gates UNDER_POWERED; observed discordance splits TOO_CLOSE/FEASIBLE. Fixture proves a counts-identical/overlap-different pair of scenarios yields DIFFERENT verdicts (the marginal-counts trap). Fixture proves the ceiling is not vacuous AND not the observed-discordance estimate (the two quantities differ on a fixture where models locate overlapping-but-discordant sets).
6. [unit] Per-pair verdict is TOTAL over the 5-member enum with the frozen predicate ORDER asserted: `PAIR_MODELS_TOO_CLOSE` distinct from `PAIR_UNDER_POWERED` on a fixture where both models locate the same cases (ceiling clears, discordance 0); `PAIR_NOT_EVALUATED_MODEL_EXCLUDED` emitted for every pair containing a preflight-excluded model, including the 14b-fails fixture voiding all three pairs.
7. [unit] `INSUFFICIENT_PILOT_EVIDENCE` fires from the FROZEN derived coverage minimum — boundary fixtures at `c=7` (fires: the 0036 first-10 subset alone) and `c=8` (does not fire); never a shaky FEASIBLE.
8. [doc] The overall fork: which pairs are `PAIR_FEASIBLE` (bake-off runs on those now) vs UNDER_POWERED / TOO_CLOSE / INSUFFICIENT / MODEL_EXCLUDED (pool enlargement, which also unblocks the 0039 A/B). No live bake-off compute spent here.
