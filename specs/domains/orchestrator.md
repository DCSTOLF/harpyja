# Orchestrator domain

Consolidated, current requirements for routing, the query classifier, the
planning matrix, the verification gate, escalation, and the citation formatter.
Each line carries its originating spec(s) as provenance; superseded text lives in
`.archive/orchestrator.md`.

- The Citation Formatter dedupes and merges overlapping spans (same-file intersecting or adjacent line ranges), ranks by `prior` + match density, breaks ties stably on `(path, start_line)`, and clamps to `max_results`. (spec 0002)
- `harpyja_locate` runs a full incremental index refresh of both `manifest.jsonl` and `symbols.jsonl` before searching (the `(mtime, size)` gate makes an up-to-date tree cost only a walk), building artifacts from scratch when none exist, so a query reflects added/modified/deleted files and their symbols with no explicit re-index call. (specs 0002, 0003)
- `max_results` is a mandatory clamp: `harpyja_locate` never returns more citations than requested. (spec 0002)
- `harpyja_locate` validates `mode`: an invalid enum value is rejected, and a valid value is accepted but surfaced via `notes` rather than silently ignored (never a silent no-op). (spec 0002)
- `harpyja_locate` honors `language_hint` best-effort by filtering candidates against the manifest's extension-derived `language`; an unrecognized hint value and the exclusion of `null`-language files are surfaced as two distinct `notes`, never collapsed into a single silent empty result. (spec 0002)
- QueryÔćĺsymbol matching is exact and case-sensitive: the query is split on whitespace into segments, each split into identifier tokens on `[^A-Za-z0-9_]`, and a symbol matches only when an identifier token equals a symbol `name` exactly (substring/fuzzy matching is not supported). (spec 0003)
- Method addressing recognizes only an ordered, adjacent `<identifier><sep><identifier>` pair (separator `.` or `::`) within one query segment, matching the left token to a symbol's `parent` and the right to its `name`; whitespace-separated identifiers and other separators do not form a method address, and a 3+ identifier chain is evaluated as every adjacent pair. (spec 0003)
- When the query matches a symbol name or a method address, that symbol's definition citation is ranked above raw ripgrep line hits (e.g. call sites) for the same token; the definition boost layers on top of the `prior` + match-density signals with the `(path, start_line, end_line, kind, name)` total-order tie-break preserved, and placeholder boost weights must preserve the asserted ordering. (spec 0003)
- When the query matches no symbol name and no method address, locate degrades to the exact ripgrep-only path and ranking ÔÇö the symbol layer is purely additive and never suppresses or reorders a plain text result. (spec 0003)
- Method addressing reuses only `.` and `::` across all languages (Rust/C++ `Foo::bar`, Java/C#/JS/TS `Foo.bar`); because `parent` is immediate-only, two same-named members under different outer types/namespaces both match the same address ÔÇö an accepted, documented ambiguity. (spec 0004)
- Routing is mode-driven via the planning matrix: `mode=auto` runs the cheapest tier that answers and climbs the seedÔćĺScoutÔćĺgateÔćĺDeep ladder only on a real signal; `mode=fast` is a Tier-1 ceiling (Scout-terminal, gate informational); `mode=deep` runs its own Tier-0 seed then skips Scout and runs Tier 2 Deep (`tiers_run=[0,2]`, `source_tier=2`, classification ignored). `harpyja_index` and `harpyja_read` make zero model calls. (specs 0005, 0006, 0008)
- The Deep seed is handed to the backend as starting citation hints (a `list[CodeSpan]` the RLM may expand via `read_span`), not as pre-loaded source content in the prompt. (spec 0006)
- A heuristic query classifier labels each query `point` or `broad` (ambiguous Ôćĺ `point`, biasing to the cheap path); the classifier is pluggable via a seam but only the heuristic ships. (spec 0008)
- The planning matrix is the single source of truth for routing, mapping `(mode ├Ś classification ├Ś index_ready)` to a planned tier ladder; the ladder executes that sequence and for `auto` the gate decides whether the final Tier-2 step runs, so realized `tiers_run` is a prefix of the planned ladder. (spec 0008)
- `index_ready` (manifest + symbol index built) is the matrix's third dimension: when false the Tier-0 seed step is skipped and the model tier runs query-only, dropping the leading `0` from the ladder; a not-ready index is a routing variant, not a floor failure (only `rg` is a hard precondition for `locate`). (spec 0008)
- The Verification Gate reads the cited lines back from disk and scores their relevance to the query via the `verify_method`-selected judge through `ModelGateway.complete()` (the only outbound caller, never a parallel judge client), with `gateway.assert_local` on the resolved endpoint before the call; `passed = score Ôëą verify_threshold`. It scores the top-`verify_top_n` ranked Tier-1 citations and logs the dropped count when the set exceeds N (no silent truncation). (spec 0008; judge selection spec 0018)
- A gate judge-call error never hard-blocks and never silently passes: the result carries the stable `gate-scoring-failed` flag and mirrors a gate-fail ÔÇö escalating to Tier-2 in `auto` when a tier remains (flag retained, `confidence=low`), else returning the best-effort un-gated Tier-1 result with the flag. (spec 0008)
- In `auto`, escalation to Tier-2 fires on any of: gate fail (top-N score `< verify_threshold`), `gate-scoring-failed`, Scout malformed (typed-bad/un-scoreable result), `broad` classification (straight to Deep, no gate), or `mode=deep`. A Scout typed-unavailable degrades to the Tier-0 floor and a Scout honest-empty returns the seed ÔÇö neither escalates. (spec 0008)
- The three Tier-1 outcomes are kept distinct: typed-unavailable Ôćĺ degrade to Tier-0 floor with `scout-degraded:<cause>` (no escalation); honest-empty Ôćĺ gate skipped, return the Tier-0 seed tagged `gate-skipped:scout-empty` (+`+no-matches` when the seed is also empty, no escalation); malformed Ôćĺ escalate to Tier-2 in `auto`. (spec 0008)
- `fast` is a caller-chosen cost ceiling: stop after Tier-1 and never escalate (even for `broad`); the gate runs informationally and, if it would have failed on scoreable citations, the Tier-1 result is returned tagged `gate-low-confidence` (emitted only when the gate actually scored). (spec 0008)
- `confidence` (`high`/`medium`/`low`) and `tiers_run` derive from the actual path taken ÔÇö terminal tier + flags, independent of whether a seed ran: gated-pass Ôćĺ `high`; honest-empty (`gate-skipped:scout-empty`) Ôćĺ `medium`/`low`; escalated-to-Deep and broad/deep straight-to-Deep Ôćĺ `medium`; degrade floor/query-only-empty Ôćĺ `low`; any `gate-low-confidence`/`gate-scoring-failed` flag Ôćĺ `low` (override). (spec 0008)
- A file-level (`None`-line) span survives the Citation Formatter without line arithmetic: it is never adjacency-merged into a lined span, sorts after lined spans of the same path on a stable key, dedups on a key carrying the `None` slot, and is returned carrying `None` lines (never crash, never fabricate a range). (spec 0011)
- A file-level (line-less) citation reaching the Verification Gate is detected before read-back and treated as a third not-verifiable outcome via `GateOutcome.skipped_reason="no-line-range"` (not scored, not a verified pass); `locate.py` attaches the stable `gate-skipped:no-line-range` marker (distinct from `gate-low-confidence`/`gate-scoring-failed`), and in `auto` with a tier remaining it escalates, else the coarse span is carried best-effort tagged with that marker, never at high confidence. (spec 0011)
- The Verification Gate judge is chosen by `verify_method` via `build_verification_gate` Ôćĺ `select_judge` (a `_JUDGE_FACTORIES` registry co-located with the factories): the default `instruct_model` builds `make_instruct_judge`, scoring relevance via `settings.lm_model` (a served, in-distribution 0ÔÇô1 instruct model) with a constrained bare-number prompt at `temperature=0`; the retained non-default `scout_model` builds `make_scout_model_judge` over `scout_model` (the finder-vs-instruct A/B baseline). This replaces the prior reuse of the OOD `scout_model` finder as the relevance judge, which false-rejected correct citations. (spec 0018)
- `_parse_score(reply) -> float | None` is strict: a `[0,1]` float only for a conforming bare score (an optional `Score:` label and a single trailing period tolerated); a bare line number (`219`, `Score: 219`), an out-of-range value (`1.2`, `-0.1`), prose-after-number (`0, becauseÔÇŽ`), or no number Ôćĺ `None`. A `None` raises a typed `ScoreParseError` (a `ValueError` subclass) so the judge degrades (`failed=True`) rather than fabricating a `1.0` pass or `0.0` reject ÔÇö preferring escalate-over-reject; both judge factories share the parse via `_score_or_raise`, and a single non-conforming reply degrades the whole `verify` call (whole-gate, not per-span). `verify`'s `except` branches on `ScoreParseError` first (it subclasses `ValueError`) and emits exactly one distinct non-conformance WARNING ÔÇö not also the generic "scoring failed" message, and distinct from the 0017 timeout WARNING. (spec 0018)
