# Deep domain

Consolidated, current requirements for Tier 2 (Deep) â€” the `dspy.RLM` over a
sandboxed REPL with bounded read-only host tools, its backend seam, air-gap
routing, layered budget bounds, and degradation. Each line carries its
originating spec(s) as provenance.

- Deep (Tier 2) is a Recursive Language Model (`dspy.RLM`) running inside a sandboxed REPL with a small set of bounded read-only host tools as its entire world; it is the strongest, most expensive tier, used only on explicit `mode=deep`. (spec 0006)
- Deep sits behind a `DeepBackend` Protocol (`run(query, seed, tools) -> list[CodeSpan]`) reached only through an injected runner (no top-level hard import), wrapped by a `DeepEngine` exposing the shared `Locator` `.search` seam returning `CodeSpan`s with `source_tier = 2`; the orchestrator/formatter never branch on `DeepBackend`, a fresh RLM instance is constructed per request, and Deep is not cached. (spec 0006)
- The RLM drives the explorer model strictly through the Model Gateway (`gateway.complete`), reusing the single `gateway.assert_local`/`AirGapError` helper with resolved-address validation before any RLM/model I/O; the Deno/Pyodide sandbox runs offline. (spec 0006)
- The RLM's entire world is four bounded read-only host tools, each a thin wrapper over Tier-0 machinery bounded by existing `Settings` caps and enforcing repo-path confinement (the RLM is an untrusted caller): `list_manifest(filter)`, `search(pattern, scope)`, `symbols(path)`, `read_span(path, start, end)`. The set exposes no mutating operation; a path resolving outside the repo is rejected and an over-budget request is clamped. (spec 0006)
- Sandboxed RLM-authored code can reach only the four whitelisted host tools â€” no ambient host filesystem and no network â€” asserted as a positive equality on the exposed binding surface and verified by an in-sandbox test (an `open()` outside or inside the repo and a raw socket attempt all fail); the residual runtime-change risk and verification method are recorded. (spec 0006)
- Explorer-loop bounds are layered so no single ignorable counter is load-bearing: externally host-enforced are `deep_max_tool_calls` (tool wrappers), `deep_token_ceiling` (Gateway), and `deep_wall_clock_ms` (a host deadline that hard-kills a non-yielding busy loop in an out-of-band host-terminable context); `deep_max_depth`/`deep_max_subqueries` are host-mediated where the runtime exposes a spawn hook, else best-effort with the recorded residual risk, transitively contained by the external ceilings. (spec 0006)
- Hitting any explorer-loop bound terminates the run with the citations gathered so far and emits a stable non-degrade note `deep-truncated:<bound>` (one of `depth`/`subqueries`/`tool-calls`/`tokens`/`wall-clock`) â€” never a `DeepUnavailable` degrade and never silent. (spec 0006)
- Deep degrades only on a typed `DeepUnavailable` (`sandbox-absent`/`rlm-down`/`backend-error`/`parse-error`) â†’ fall back to Scout best-effort with a stable `deep-degraded:<cause>` note (double-degrading to Tier 0 if the model is also down); it never degrades on a quality heuristic â€” a successful run returning few/zero citations is an honest Tier-2 result (`tiers_run=[0,2]`), not a fallback. `RipgrepMissingError` and `AirGapError` propagate loudly. `parse-error` is a **sibling** of `backend-error` (a named, narrow-caught, pinned parse failure â€” a dspy `AdapterParseError` at the `rlm(query=...)` seam mapped `raise DeepUnavailable(PARSE_ERROR) from err`), **not** a replacement: truly-unexpected exceptions still fold to `backend-error`, and the typed-failure-only boundary is unchanged (a well-formed-but-weak/empty Deep response stays an honest Tier-2 result, never a degrade). (specs 0006, 0014)
- The `parse-error` catch is narrow and pinned: catch `dspy.utils.exceptions.AdapterParseError` (the single class all four dspy adapters raise â€” raw JSON decode is wrapped into it â€” pinned against dspy 3.2.1 source) **alone**, resolved lazily so the module keeps its no-top-level-`import dspy` rule (`except ()` catches nothing when dspy is absent); wrap **only** the `rlm(query=...)` forward call â€” `_assert_local` (the `AirGapError` floor) and `_rlm_factory` (config faults) stay outside the catch, and `parse_citations` never raises. (spec 0014)
- Deep `search` on a nonexistent scope returns the typed model-visible marker `search-scope-not-found: '<scope>'` (an `exists()` guard added before engine delegation, `_charge()` kept FIRST since a bad-scope call is still a real charged tool call) â€” closing a previously UNCAUGHT `FileNotFoundError` from `subprocess.run(cwd=<nonexistent>)` on the `host_tools.search`/`RlmBackend.run` path that violated the graceful-degradation guardrail; its existing FILE-scope engine delegation (real repo-relative results since 0033) is pinned as the behavior the explorer `grep` now converges to (spec 0035)
