# Gateway domain

Consolidated, current requirements for the Model Gateway and the air-gap
boundary. Each line carries its originating spec(s) as provenance.

- `ModelGateway.assert_local()` passes for `127.0.0.0/8` and `::1` endpoints (or hosts resolving only to those) and rejects any non-loopback endpoint unless `allow_remote` is set, making no network call. (spec 0001)
- The Model Gateway is the single outbound caller â€” a thin client over a local OpenAI-compatible endpoint (llama.cpp `llama-server` or Ollama) holding the base URL + primary model from `Settings`. (spec 0005)
- `gateway.assert_local` takes an injected resolver (stdlib default, fake in tests), resolves the configured endpoint host to its address set, and asserts every resolved address is loopback via the stdlib `ipaddress` predicate; `localhost` is accepted only because it resolves to loopback. No request is ever sent to a non-loopback address. (spec 0005)
- A configured non-loopback endpoint raises a typed, actionable `NonLoopbackEndpointError`/`AirGapError` naming the offending host â€” a loud floor case, never a silent skip and never a degrade note. (spec 0005)
- The Model Gateway's single outbound HTTP call is time-bounded: `_default_transport` calls `urlopen(req, timeout=timeout_s)` (previously unbounded â†’ a stalled/torn-down local endpoint could wedge forever). The timeout rides on a `ModelGateway.timeout_s` field whose dataclass default is itself finite (`120.0`, never `None`) so any `ModelGateway(...)` construction is hang-bounded, and it is bound onto the default transport via `functools.partial` only when `transport is None`, leaving the injected two-arg `Transport` seam signature untouched. It is a per-socket-op bound, not a total-request deadline (a dribble-slow endpoint is out of scope). A stall now raises `TimeoutError`/`socket.timeout`/`URLError` that propagates out of `complete()` (never swallowed inside the gateway), so the Verification Gate's existing `except â†’ GateOutcome(failed=True)` degrades gracefully instead of hanging, emitting a distinct timeout-naming WARNING that separates a judge timeout from other judge failures. (spec 0017)
- `ModelGateway.complete_with_tools(messages, tools, *, transport, resolver, **params)` returns `{content, tool_calls, finish_reason, model}` for tool-calling models (the Scout explorer loop) â€” `finish_reason` reads `choices[0].finish_reason` (the CHOICE not the message, `str`-cast, defaulting to `"unknown"` when absent) and `model` surfaces `response.get("model")` (the served-model identity for trajectory verification), both backward-additive; the same single outbound abstraction, `assert_local` asserted BEFORE any transport touch, transport injectable exactly like `complete`, and `ModelGateway` stays purely param-driven (no `max_tokens` default of its own, so the Deep-tier path is never capped). (specs 0024, 0028, 0031)
