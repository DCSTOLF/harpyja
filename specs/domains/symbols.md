# Symbols domain

Consolidated, current requirements for the symbol layer (tree-sitter extraction,
the ripgrep engine, grammar coverage, kind vocabularies, artifact reproducibility,
and degradation). Each line carries its originating spec(s) as provenance;
superseded text lives in `.archive/symbols.md`.

- `RipgrepEngine.search` returns bounded `CodeSpan`s confined to `repo_path`, honoring `search_max_files`, `search_max_matches`, and `rg_chunk_size`; the query is a literal string by default (regex metacharacters matched literally) and a single-line match yields `start_line == end_line` with no surrounding context. (spec 0002)
- `rg` on PATH is required for search/locate only; when absent, `harpyja_locate` and search fail with a clear, typed, actionable error naming ripgrep (never a silent empty result) and `doctor` reports its absence. (spec 0002)
- The SymbolEngine extracts definitions only (via tree-sitter) into `symbols.jsonl` ŌĆö one JSON object per definition with `path` (repo-relative), `language`, `name`, `kind`, `parent` (immediately enclosing type/class, or `null` for top-level), `start_line`, `end_line` (1-indexed inclusive, matching `harpyja_read`/`CodeSpan`). (spec 0003)
- The kind vocabulary is a closed, documented set classified by syntactic form (no type/value inference); Python emits `function`, `method`, `class`, module-level `constant` (single `UPPER_SNAKE` target, tuple/augmented assignment excluded); Go emits `function`, `method`, `struct`, `interface`, `type`, package-level `const`/`var`, with pointer/value/generic receivers normalized so `parent` is the bare type name. (spec 0003)
- Imports, references/call sites, and function-local (nested) definitions are not extracted ŌĆö `symbols.jsonl` holds top-level and class/struct-member definitions only, not a usage graph. (spec 0003)
- Symbol extraction never fails the index: a grammar that can't be imported/loaded flags every file of that language `grammar-missing` with zero records (ripgrep unaffected); a parse error skips only a definition whose own region (excluding nested-definition subtrees) carries an `ERROR`/`MISSING` node while emitting clean siblings, and flags the file `parse-error` whenever its tree has any `ERROR`/`MISSING` node. The two causes are distinguishable and name the language/file. (spec 0003)
- `harpyja_index` writes `symbols.jsonl` alongside `manifest.jsonl` in the same artifact dir; `symbols_indexed` reports the total record count in-index after the refresh (not parsed-this-run), so an incremental refresh that re-parses nothing still reports the full total. (spec 0003)
- Each file's degradation outcome (`grammar-missing`/`parse-error`/clean) is persisted on its `manifest.jsonl` entry and reused with the entry across the `(mtime, size)` gate, so an incremental refresh that re-parses nothing still returns the full, accurate `degraded` array (total-in-index); pruning drops the field with the entry. (spec 0003)
- Symbols are re-extracted only for files whose `hash` changed; a file passing the `(mtime, size)` gate reuses its prior symbol records without re-parsing, and `--rehash` re-parses every file. (spec 0003)
- `symbols.jsonl` is paired with a byte-reproducible `symbols.meta.json` sidecar carrying `schema_version`, `engine_identity` (tree-sitter runtime + each pinned grammar version, with a `"missing"`/`"load-error:<abi>"` sentinel for an absent/unloadable grammar), `record_count`, and a `content_digest` (`"sha256:ŌĆ”"`) over the exact `symbols.jsonl` bytes. (spec 0003)
- A full symbol rebuild (independent of the `(mtime, size)` gate and `--rehash`) is forced whenever `symbols.jsonl` is missing/unreadable/truncated, the sidecar is missing/unreadable while records exist, the meta's `engine_identity` (including a grammar-only version bump or an absentŌåÆpresent grammar) does not match the running engine, or the records' line count/`sha256` do not match the sidecar's `record_count`/`content_digest`. An absentŌåÆpresent grammar rebuild also clears stale `grammar-missing` flags for now-parseable files. (spec 0003)
- The two symbol artifacts are committed records-first then meta-last, each via a same-directory temp file + `os.replace`; a crash leaving fresh records under a stale meta is caught by the fingerprint mismatch and rebuilds (the reverse order is forbidden). (spec 0003)
- When a tracked file is deleted or renamed, a refresh removes its symbol records so no `symbols.jsonl` line cites a path no longer on disk. (spec 0003)
- Two indexes of an unchanged tree produce a byte-identical `symbols.jsonl` and `symbols.meta.json` (records ordered by the total key `(path, start_line, end_line, kind, name)`; sidecar with fixed key order and a stable-sorted `languages` set of the distinct languages with Ōēź1 record). (spec 0003)
- The build pins each tree-sitter grammar package individually at an explicit version (Python, Go, Rust, Java, C#, JavaScript, TypeScript+TSX via one `tree-sitter-typescript` package, C, C++) plus the `tree-sitter` runtime ŌĆö never the aggregate `tree-sitter-languages` wheel; `engine_identity` enumerates every shipped grammar; parsing runs fully locally with bundled grammars and introduces no runtime network egress, leaving `gateway.assert_local` and the inbound-bind defaults untouched. (specs 0003, 0004)
- The SymbolEngine implements the shared `Locator` protocol and returns the common `CodeSpan`/`Citation` shapes; the orchestrator composes it with the ripgrep `Locator` and never branches on whether a symbol engine is present. (spec 0003)
- A language's extension routing, its `engine_identity` slot, and its kind-extraction rules ship in the same change (no-silent-coverage invariant): until a language ships, its extensions stay unmapped (`null`-language/ripgrep-only) and its grammar is absent from `engine_identity`, so partial coverage degrades visibly, never as silent zero-symbol coverage. (spec 0004)
- Symbol grammars cover Rust, Java, C#, JavaScript, TypeScript, TSX, C, and C++ via individually pinned tree-sitter packages, with extensionŌåÆlanguageŌåÆgrammar routing: `.rs`ŌåÆrust, `.java`ŌåÆjava, `.cs`ŌåÆcsharp, `.js`/`.mjs`/`.cjs`/`.jsx`ŌåÆjavascript, `.ts`ŌåÆtypescript, `.tsx`ŌåÆtsx, `.c`/`.h`ŌåÆc, `.cc`/`.cpp`/`.cxx`/`.c++`ŌåÆcpp and `.hpp`/`.hh`/`.hxx`/`.h++`ŌåÆcpp; `.h` defaults to the C grammar (documented ambiguity) and `.mts`/`.cts` stay unmapped. (spec 0004)
- Per-language kind vocabularies (minimal, syntactic-form only, definitions only): Rust emits `function`/`method`/`struct`/`enum`/`trait`/`type`/`constant` (`const`+`static`, `UPPER_SNAKE` module-level only, impl/generic receivers normalized to the bare type); Java/C# emit type kinds (`class`/`interface`/`enum`, plus C# `struct`) and `method`; JS/TS/TSX emit `function`/`method`/`class`/`constant` (single `UPPER_SNAKE` `const`, `export const` included), with TS/TSX additionally `interface`/`type`/`enum`; C emits `function`(with body)/`struct`/`enum`/`union`/`type`; C++ emits `function`/`method`/`class`/`struct`/`enum`/`union`/`type` with out-of-line `Foo::bar` normalized to `parent` `Foo`. (spec 0004)
- A type defined directly inside another type body (nested class/struct/interface/enum) IS extracted with `parent` = the immediately enclosing type name, and its methods record that inner type as `parent`; only function/method-body-local definitions (local classes, closures, local functions) are not extracted. (spec 0004)
- `RipgrepEngine.search` takes an optional per-CALL `repo_root=` keyword (not a constructor field ŌĆö the shared engine is built once across repos): supplied, a directory-scope search emits REPO-RELATIVE paths via a parse-side re-prefix (mechanism b ŌĆö `_parse(rel_prefix=ŌĆ”)` over `normpath(join(rel_prefix, path))`) while the real `rg` invocation stays BYTE-IDENTICAL (same `cwd=scope`, flags, ignore-file, ordering), and a FILE scope runs `rg` from the file's PARENT with the filename as an rg path arg (net-fixing the pre-existing `NotADirectoryError` crash in the `symbols` degraded fallback); the legacy no-`repo_root` path is verbatim-unchanged (Tier-0 untouched). (spec 0033)
- Path-shape defects are fixed at the ONE `RipgrepEngine` seam: every path-DISCOVERING consumer (explorer `grep`, the `symbols` degraded fallback, Deep `search`) supplies `repo_root` as DATA ŌĆö never re-prefixing per-caller, never repairing downstream. (spec 0033)
