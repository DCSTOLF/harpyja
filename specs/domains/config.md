# Config domain

Consolidated, current requirements for layered settings and config discovery.
Each line carries its originating spec(s) as provenance.

- Settings resolve with precedence profile defaults < `harpyja.toml` < `HARPYJA_*` env < per-request override (lowest to highest). (spec 0001)
- `harpyja.toml` discovery order is explicit `--config <path>` > cwd `harpyja.toml` > repo-root `harpyja.toml`. (spec 0001)
- Scout budgets are `Settings`-configurable: `scout_seed_top_n` (default 5), `scout_max_citations` (default 20, clamped to `min(scout_max_citations, max_results)`), `scout_max_span_lines` (default 200). (spec 0005)
- Deep budgets are `Settings`-configurable: `deep_seed_top_n` (5), `deep_max_citations` (20, clamped to `min(deep_max_citations, max_results)`), `deep_max_span_lines` (200), `deep_max_depth` (3), `deep_max_subqueries` (8), `deep_max_tool_calls` (200), `deep_token_ceiling` (32000), `deep_wall_clock_ms` (60000). (spec 0006)
- `scout_model` selects the Scout/FastContext fine-tune (default the served Ollama GGUF `hf.co/dstolf/FastContext-1.0-4B-RL-Q8_0-GGUF:latest`, flipped from the unserved `mitkox/…RL-Q4_K_M` tag that 404'd on every Scout call), distinct from Deep's `lm_model`, with standard precedence (defaults < `harpyja.toml` < `HARPYJA_*` < per-request). (specs 0007, 0016)
- `lm_model` (Deep's Tier-2 explorer model) defaults to the served `hf.co/Qwen/Qwen3-8B-GGUF:latest` (provisional, "for now"), flipped from the llama.cpp placeholder `"local"`; this is a global `Settings` default so it affects every bare `Settings()` caller — the eval `run`/`sweep` drivers, the MCP server's `mode=auto` Deep tier, and (as of spec 0025) the Scout explorer, whose canonical `build_scout_engine` default gateway pins `settings.lm_model` because the old `ModelGateway.model` `"local"` default 404s on Ollama's tag-routed API, so Scout and Deep now run the SAME model; a llama.cpp operator (for whom the Ollama-style tag will not resolve) must set `lm_model` explicitly via toml/env/`--deep-model`. (specs 0016, 0025)
- `lm_http_timeout_s` (the gateway outbound-HTTP timeout, `float` seconds) defaults to a finite positive `120.0` (never `None`), coerced from toml/env like the other numerics with precedence defaults < `harpyja.toml` < `HARPYJA_*` and no per-request layer (it is fixed at gateway construction, not per `complete()` call); deliberately decoupled from `deep_wall_clock_ms` (60 s) — they bound different things (a per-socket-op HTTP wait vs. a total Deep budget). (spec 0017)
- Verification Gate settings (appended last with defaults, standard precedence): `verify_method` (default `"instruct_model"`; `_VERIFY_METHODS = {"scout_model", "instruct_model"}` — an unsupported value is rejected loudly at `Settings` load with a typed `UnsupportedVerifyMethod`, never a silent fall-through; the default selects the instruct-model judge scoring via `lm_model`, so `lm_model` backs BOTH Deep Tier-2 and the gate judge — one Settings value, two subsystems — while the retained `scout_model` selects the finder judge over `scout_model`), `verify_threshold` (provisional `0.6`), `verify_top_n` (provisional `3`). (specs 0008, 0018)
- `scout_ls_max_entries` (default 200) clamps the explorer's `ls` single-directory listing — an additive-last `Settings` field under standard precedence. (spec 0027)
- `explorer_max_tokens` (default 2048, the per-call anti-runaway generation cap) and `explorer_enable_thinking` (default `True`) are additive-last `Settings` fields passed ONLY by the explorer's model call; the `explorer_`-prefix scopes them to Scout by construction so the Deep-tier outbound call carries neither (a guard asserts the actual outbound fields, not the Settings names). (spec 0028)
- `scout_symbols_max_entries` (default 400) clamps the explorer's `symbols` tool output — an additive-last `Settings` field. (spec 0030)
- `explorer_think: bool | None = None` is an additive-last, `explorer_`-scoped `Settings` field (tri-state: `None` ⇒ omit the `think` param ⇒ outbound request byte-identical to pre-knob, `True/False` ⇒ send `think=that`), passed ONLY by the explorer with the Deep-tier outbound guard extended to carry neither the knob nor the param; `_coerce` is extended for `bool | None` env coercion (spec 0034)
