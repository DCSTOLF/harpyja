# Eval domain

Consolidated, current requirements for the evaluation harness — measurement-only,
recommend-only — including the dataset formats, the overlap oracle, the
threshold/top-n sweep, SWE-bench adaptation, and degrade-visibility reporting.
Each line carries its originating spec(s) as provenance.

- The eval harness is measurement-only and recommend-only: it observes the real `mode=auto` path without modifying tier/orchestrator/gate/matrix behavior and emits recommendations (e.g. an OQ2 trade-off table) without flipping any `Settings` default — default flips are separate follow-up specs citing the evidence. (spec 0009)
- Eval-only knobs (repeated-run count `K`, proximity window, N-floor, catch-rate bar, `degraded_dominated_threshold`) live on a dedicated `EvalConfig` dataclass in `harpyja/eval/`, field-disjoint from the production frozen `Settings`; the only `Settings` interaction is the sweep building grid points via `dataclasses.replace`, never mutation. (spec 0009)
- The eval dataset format is per-case `{query, repo, expected file:line span(s), classification label}` stored as a versioned fixture; the loader rejects malformed cases loudly with a typed, actionable error, never a silent skip. (spec 0009)
- Span-hit accuracy is line-range overlap as the primary metric, with file-level + proximity as a secondary looser metric reported alongside; a single overlap oracle defines correctness and is reused for both gate catch rate (wrong Tier-1 → escalated) and gate false-escalation rate (correct Tier-1 → wrongly escalated). (spec 0009)
- Each measured configuration runs `K` times and the report carries mean + spread per metric; a configuration is recommended over another only when its advantage exceeds the observed run-to-run variance. (spec 0009)
- Sweep mode runs the case set across a `verify_threshold` × `verify_top_n` grid (each point via `dataclasses.replace`, `K` runs per point) and reports the accuracy/escalation/cost trade per point; an N-floor below which results are flagged indicative-only is stated in the report. (spec 0009)
- The runner emits a JSON + human report conforming to a pinned schema (stable per-case and aggregate field names); report artifacts are written to a temp/output directory outside any indexed/target repo (read-only guardrail), and the run path is fully offline (zero non-loopback egress). (spec 0009)
- The SWE-bench Verified dataset adapter uses the standalone localization protocol (no Docker, no patch application, no test execution): it queries the pre-fix repo at `base_commit` and scores predicted citations against patch-derived pre-image hunk spans. Absolute accuracy is treated as contamination-caveated; the load-bearing outputs are relative deltas (sweep and `fast`-vs-`auto`). (spec 0010)
- The dataset pipeline (`harpyja/eval/swebench_eval.py`) separates stages by network posture: `convert` (HuggingFace) → committed portable `swebench_verified.raw.jsonl`; `provision` (may `git clone`) → gitignored machine-local `swebench_verified.resolved.jsonl` with `repo` rewritten to a worktree at `base_commit`; `prune`; and offline `run`/`sweep` subcommands. `convert`/`provision` are dev-time tools outside the runtime air-gap guarantee; `run`/`sweep` are offline. (spec 0010)
- Query classification for SWE-bench cases is derived from patch shape (single file with small total pre-image span → `point`; multi-file or large-span → `broad`), and the driver overrides production routing to that label via the `LocateStack.classifier` injection seam so the gate fires on the point subset; each per-case event records both the patch-shape label and the production `classify_query` label and the aggregate reports their agreement rate, with the OQ2 recommendation flagged low-confidence (deltas-only) below a named agreement floor. (spec 0010)
- Cases whose gold-patch targets are all new files (`--- /dev/null`) are flagged `new_file_only` and excluded from span/file scoring (the excluded count is a durable report field), never scored as a silent zero; a pure-insertion hunk in an existing file is anchored to a concrete one-line span and is scored. The pre-image hunk oracle's context-line inflation biases span-hit upward and is recorded as a tolerance. (spec 0010)
- A per-case-repo driver builds a fresh `LocateStack` per case using that case's worktree as `repo_path` and pools outcomes into the aggregate via the unchanged overlap oracle and recommend scorer; the report schema is extended additively (fields appended last with defaults, `SCHEMA_VERSION` bumped) so both single-run and multi-repo shapes validate, and artifacts are written via `atomic_write_json` outside every case repo. (spec 0010)
- `run_case` threads `mode` so the driver can run the set `mode="fast"` (a Scout-terminal no-climb Tier-1 line where the gate runs informationally) alongside `auto`; the report carries durable metadata fields including protocol identity, excluded/malformed counts, classifier-agreement rate, span-inflation tolerance, contamination caveat, dataset provenance (HF id/split/revision, raw-fixture hash, sample case-ids), and a SUT-observed `production_gate_ran` flag kept distinct from the harness-observed `gate_triggered` probe. (spec 0010)
- The harness reports degrade visibility as first-class machine-readable fields: `scout_degrade_count` and `scout_degrade_rate` (= count / cases_attempted, explicit `null` paired with a zero count when the denominator is zero), `degraded_dominated` (rate > `degraded_dominated_threshold`, provisional eval-only `0.5`) with composable `reliability_notes` stable identifiers, and FC text-ref shape counters `fc_citation_{spanned,filelevel,dropped}_count`. (spec 0011)
- The overlap oracle scores a file-level (line-less) citation as a path-only match recorded distinctly from a line-overlap match (a coarse hit is never a line hit), taking the path-only branch before the line arithmetic; a file-level cited span serializes its line fields as JSON `null` and `validate_report` tolerates null cited lines (gold `expected_spans` stay int-only). (spec 0011)
- Recovery observability is plumbed producer→sink: `build_scout_engine` loads the manifest file list into `ScoutEngine` (empty set for unit fakes / index-not-ready → no recovery), both `eval/runner.py` and `eval/swebench_eval.py` aggregate the per-case `last_tally` counts, and `fc_citation_recovered_{spanned,filelevel}_count` are appended last-with-default (`SCHEMA_VERSION` `0012/1`, later additively bumped to `0013/1` for the `deep_degrade_count`/`deep_degrade_rate` twins and union-based `degraded_dominated`, validator accepts legacy and new shapes). (specs 0012, 0014)
- Every typed-degrade floor reports a paired `<tier>_degrade_count`/`<tier>_degrade_rate` twin (additive, appended last-with-defaults, explicit `null` paired with a zero count on a zero denominator) **and** feeds `degraded_dominated`, which keys off the **union** of all tiers' per-case degrades — a case counted **once** even when multiple tiers floor — with the per-tier rates kept separate for attribution; a shared `_has_degrade_note` predicate backs `_is_scout_degraded`/`_is_deep_degraded`. Deep's `deep_degrade_count`/`deep_degrade_rate` (report schema `0013/1`) is the first additional instance after Scout. (spec 0014)
- `cmd_provision` (`eval/swebench_eval.py`) resolves `--work-dir` to an absolute path (`Path(args.work_dir).resolve()`) before `git worktree add`, so a relative work-dir produces worktrees whose real location matches the resolved `repo` path recorded in the machine-local fixture — a relative dir otherwise created trees under the clone cwd while the fixture recorded `wt.resolve()`, so every resolved repo path 404'd. (spec 0015)
- The `run`/`sweep` parsers expose model overrides via the shared `_add_model_flags` group: `--scout-model` (→ `scout_model`) and the canonical `--deep-model` (→ `lm_model`), with `--lm-model` retained as a deprecated back-compat alias; the two Deep flags get distinct argparse dests reconciled in `_settings_from_args` so `--deep-model` wins regardless of CLI order, and every override is built via `dataclasses.replace` on a fresh `Settings()`, never mutating the frozen base. (spec 0016)
- The OQ2 recommender refuses to calibrate over a gate it has measured to be broken: `EvalConfig.gate_false_escalation_ceiling` (eval-only, `Settings`-disjoint, provisional `0.20`) is the gate-confound bar, and `recommend_oq2(points, measured_false_escalation, eval_config)` is a dispatcher wrapping the unchanged `rank_sweep` — a measured instruct-judge false-escalation **strictly `> ceiling`** returns the `gate-confounded` typed null (`OUTCOME_GATE_CONFOUNDED`, carrying the measured rate) rather than tuning `verify_threshold` over a judge that rejects correct citations; `== ceiling` is not confounded (strict `>`) and an unmeasured `None` defers. `Recommendation.outcome`/`.gate_false_escalation_measured` are additive (appended last-with-defaults). (spec 0019)
- `preflight_models_present(settings, tags_payload, *, resolver=None)` is a setup-time doctor (`PreflightError` + `cmd_preflight` + a `preflight` CLI subparser): it asserts `gateway.assert_local` on the loopback endpoint **first** (no second outbound path — the `/api/tags` read is the same loopback-gated egress class it preflights), then checks a **deduped required-tag set** (`_required_model_tags`: `scout_model` ∪ `lm_model`, since deep-model and the instruct judge both resolve to `lm_model`) is served, raising a typed error **naming** the first absent tag. It claims only that models are **pulled**, NOT co-resident-loadable — OOM under `mode=auto` is named as a residual mid-run risk the cheap G1 smoke catches (no-false-capability). (spec 0019)
- The eval report schema is `0014/1` (additive last-with-defaults over `0013/1`): run-metadata `gate_false_escalation_ceiling`; aggregate `gate_confounded`/`gate_confounded_measured_rate` + instruct/scout A/B false-escalation twins (`gate_false_escalation_{instruct,scout}` + `_count`/`_total`, null-with-zero-count), hoisted to one `_GATE_CONFOUND_AGG_FIELDS` anti-drift source with a drift-guard test; legacy `0013/1` blocks still validate. `run_swebench_sweep` wires `recommend_oq2` in (best-achievable instruct false-escalation = min over measured grid points, only when the base `verify_method == "instruct_model"`) and stamps the ceiling into metadata + `outcome`/`gate_false_escalation_measured` into the recommendation block — the gate-confound outcome is reached by the real driver, not left wired-but-dormant. (spec 0019)
- The OQ2 G3 outcome taxonomy is a **projection layer above the byte-frozen `recommend_oq2`**: `classify_g3_outcome(recommendation, aggregate, eval_config)` (in `harpyja/eval/oq2_classify.py`) maps to exactly one of {`RECOMMENDATION`, `GATE_CONFOUNDED`, `DEGRADED_DOMINATED`, `NOT_SEPARABLE`} by the total order `DEGRADED_DOMINATED > GATE_CONFOUNDED > NOT_SEPARABLE > RECOMMENDATION`, recording ALL true blocking conditions (not just the winner). The no-survivor signal `S = (incumbent_validated is False AND advantage_exceeds_variance is False)` is read from the frozen `Recommendation` and computed only when the outcome is not `gate-confounded` (else n/a — no phantom `NOT_SEPARABLE`); `indicative_only` is a `RECOMMENDATION`-only sub-flag (effective_N < `n_floor`). The dispatcher/`rank_sweep` stay byte-frozen, pinned by a field-reachability lock and a behavior snapshot. (spec 0020)
- The OQ2 operator run is a sequential G0→G1→G2→G3 stop-and-report protocol (`harpyja/eval/oq2_protocol.py`, driven live by `oq2_live.py` + the `oq2` CLI) that records each gate verdict before the next runs and emits a NEW pinned **gate-ledger** (`LEDGER_SCHEMA_VERSION = "0020/1"`, distinct from report `0014/1`, loudly validated, written via the shared `atomic_write_json`). Close-vs-hold is drawn **by cause**: G0 preflight fail, fixtures absent, or a G1 sub-check-(a) non-completion for an environment reason (OOM/resource) → `BLOCKED` **hold**; a run that completed then degrade-dominated or gate-false-rejected → `STOP:SMOKE` **close**; a G3 label closes. A measurement spec closes only on a recorded SUT-observing typed outcome — skip-not-fail because the environment is absent is **never** a valid close. (spec 0020)
- A measurement spec **verifies a typed null is real (not a harness artifact) before reporting it** and names the actual upstream blocker: 0020 live run produced a typed **DEFERRED** — Scout Tier-1 = 0 correct on the SWE-bench point subset, so `gate_false_escalation` is `null` by definition and the finder A/B + G3 sweep are moot. Verified by direct Tier-1 spot-checks (`expected_spans` load, Scout runs/returns citations, oracle correct — right-file-wrong-span at best, e.g. astropy-12907 `separable.py:66-102` vs gold `242-248`) and cross-checked with an independent finder-model A/B (`qwen3:4b-instruct` as Scout also 0/3, identical wrong span on astropy → span-level localization / task difficulty, not model quality). Named follow-up: **Scout Tier-1 span-level localization on SWE-bench**, NOT gate-accuracy / threshold / finder-swap. (spec 0020)
- The escalation-trigger classifier (`harpyja/eval/escalation.py`) is a **pure projection over the byte-frozen `_locate_auto` escalation decision**: `WrongCitationFate` {`GATE_FALSE_ACCEPTANCE`, `NO_ESCALATION_PATH`, `DEEP_DEGRADED_OR_UNAVAILABLE`, `NOT_APPLICABLE`} + `classify_escalation(*, tier1_correct, gate_rejected, deep_available, ladder, tier1_empty=False) -> (will_escalate, fate)`; the ladder is passed IN (the classifier never re-derives `matrix.plan_ladder`; the trigger test obtains every ladder BY CALLING it, guarded equal). Empty Tier-1 → `NO_ESCALATION_PATH` (honest-empty is gate-skipped and never escalates, per frozen `_locate_auto`), gate-passed-wrong → `GATE_FALSE_ACCEPTANCE`, gate-rejected-but-Deep-down → `DEEP_DEGRADED_OR_UNAVAILABLE`. Additive; SUT (`harpyja/orchestrator/`) read-only reference, not modified. (spec 0021)
- The instrumented ≤2-case escalation micro-run (`harpyja/eval/escalation_microrun.py`) attributes per-tier wall-clock at the **eval boundary** by wrapping collaborators' public methods (`_wrap_timed` around `scout_engine.search`/`gate.verify`/`deep_engine.search`, a safe no-op on a None/missing method, restored in `finally`) — never inside frozen orchestrator internals — and reproduces `escalation_rate` from `tiers_run`; the per-tier split is labeled `"estimate"` (the frozen runner persists only case-level `latency_ms`, not Scout/judge/Deep granularity), the total stays wall-clock-anchored, and `build_micro_result` flags `deep_degraded`. Integration test is skip-not-fail (served-model/fixture-gated). (spec 0021)
- A metric-integrity diagnostic **pins the existence of its per-run dump before relying on it** (`eval_work/` is gitignored / machine-local, so per-run diagnostics evaporate and un-committed secondaries survive only in an operator transcript); if the dump is absent the time-attribution degrades to an explicitly-**labeled estimate**, never a fabricated recorded number. The deliverable is a **MECE two-axis typed finding** (`accounting` × `wrong_citation_fate`, because a flat enum is not mutually exclusive) plus a **metric-trust verdict** naming which prior secondaries are trustworthy vs contaminated (regenerate, don't inherit). Spec 0021 proved the 0020 `escalation_rate=0`-vs-3.3h anomaly is `accounting = CORRECT_NO_ESCALATION` (the metric is derived from `tiers_run`, coupling-pinned — not a counter bug), the 3.3h sink is Scout FastContext exploration (not Deep), and flagged `wrong_tier1_count`/`span_hit_rate_primary`/`gate_catch_rate` CONTAMINATED; the 0020 DEFERRED verdict is unchanged. (spec 0021)
- The Scout locate-accuracy taxonomy (`harpyja/eval/locate_accuracy.py`) is a **pure projection over the byte-frozen overlap oracle** (`metrics.span_hit_kind`/`span_hit_secondary`, untouched): `normalize_citations` reads the spec-0012 `ScoutTally` recovery/drop counts (never re-derives suffix recovery, surfaces `normalization_dropped` so EMPTY-after-drop stays distinct from returned-nothing); a 4-way MECE `LocateBucket` {`EMPTY`, `WRONG_FILE`, `RIGHT_FILE_WRONG_SPAN`, `CORRECT`} with strict precedence `CORRECT > RIGHT_FILE_WRONG_SPAN > WRONG_FILE > EMPTY` takes each case's single best bucket over all its citations. The ONE deliberate scored departure from the oracle — a path-only right-file hit (`span_hit_kind == "file"`) re-mapped to `RIGHT_FILE_WRONG_SPAN`, not the oracle's coarse primary-hit — is the file-vs-span diagnostic axis and lives ONLY in the eval classifier (`metrics.py` untouched), double-guarded by a `SUT_SURFACE` frozenset allowlist of the exact frozen public names the module may touch + a frozen-oracle input→output behavior snapshot. (spec 0022)
- A locate diagnostic scores at **two granularities with the gap first-class**: `score_distribution` → `LocateDistribution` reports file-level accuracy (`|CORRECT ∪ RIGHT_FILE_WRONG_SPAN| / n`), span-level accuracy (`|CORRECT| / n`), and `gap = file − span` as a reported field — the gap IS the discriminator (large gap → precision fix; low file-level → capability/retrieval fix), never conflated into one accuracy number. `decide_finding` routes over PRE-DECLARED named bands (`LOW_FILE_BAND`/`EMPTY_DOMINANT_BAND`/`LARGE_GAP_BAND`/`MATERIAL_DELTA_EMPTY`) to exactly one of `BENCHMARK_UNREPRESENTATIVE > PRECISION_FIXABLE > RETRIEVAL_FUNDAMENTAL > MIXED` (ordered, all true conditions recorded, each label naming the fix/next-spec), with a pre-registered prior recorded before the run as a falsifiability guard. (spec 0022)
- The Scout-only locate probe (`harpyja/eval/locate_probe.py`) drives `scout_engine.search` in isolation (no gate/judge/Deep), resets `last_tally` per case, and REGENERATES the distribution from scratch (never inherits 0021's contaminated counts); `stratify_cases` keys strata by repo × gold-span-size band, `run_reformulation_probe` records a raw-vs-distilled empty-rate delta held OUT of the baseline. Availability is tier-scoped (`scout_stack_available` = fastcontext + `rg` + reachable Scout endpoint; NOT the Deep-oriented `_live_stack_available`, which false-skips a Scout-capable host on absent Deno), and the fail-posture is split (`require_live_stack` + `HARPYJA_REQUIRE_LIVE_STACK`: integration test files skip-not-fail, the deliverable run fails loud so a skip never counts as a measurement). (spec 0022)
- The benchmark-fit verdict machinery (`harpyja/eval/benchmark_fit.py`) is **pure** (no SUT import, no I/O): an exact two-sided McNemar from `math.comb` (`mcnemar_exact_p`/`mcnemar_rejects`, boundary-pinned 6/0→0.03125 & 8/0→0.0078125 reject, 5/0→0.0625 & 7/1→0.0703125 don't); a frozen `PREREGISTERED_CONFIG` (`MIN_DISCORDANT_PAIRS=8` derived from exact-McNemar reachability — under H0 the discordant pairs are a sign test at p=0.5, so α=0.05 is first clearable at `n_discordant≥6` and 8 buys one contrary pair of slack; a round 5 was wrong because it makes the positive verdict structurally unreachable — `DELTA_EMPTY_BAND=0.20`, `min_n=12`, `alpha=0.05`); `PairedRow`+`aggregate_paired` computing within-case `delta_empty`/`delta_file_accuracy` + discordant `(b,c)` FROM retained pairs (never a difference of aggregate rates). The verdict is a total PURE FUNCTION over the frozen config so it cannot be steered post-hoc, and the mechanical rule / LLM prompt are hashed before any run. (spec 0023)
- Axis 1 (query shape) is a **within-case paired A/B over a binary outcome**, so its power lives in the McNemar **discordant pairs** — the effective sample size is the discordant (flip) count, not N (reaching 8 flips may need ~15–25 raw cases; the config states that honest cost). Total `decide_axis1` applies a paired uncertainty gate with three **named, non-overlapping** `INCONCLUSIVE` triggers (`INSUFFICIENT_POWER`/`DISTILLER_ARM_DISAGREEMENT`/`AXIS_SIGNAL_DISAGREEMENT`); Axis 2 is a structured `RepresentativenessRecord` (query-shape, repo-type, documentation-density, codebase-age, target-proxy-validity) + `is_representative` (`false` iff documentation-density=low AND target-proxy-validity=weak). `compose_verdict` is a total, pre-registered 2×2 over `Axis1Verdict × bool` where Axis 2 can **downgrade** Axis 1's routing (`QUERY_SHAPE×¬representative` → build a terse-query benchmark first, **NOT** a finder swap; `CAPABILITY×¬representative` → retire SWE-bench; `INCONCLUSIVE` → hold) — the cell names the next spec; totality pinned by grid tests. (spec 0023)
- The distiller (`harpyja/eval/distill.py`) is **dual with asymmetric roles**: `mechanical_distill` is PRIMARY/verdict-driving — a single case-agnostic, gold-span-blind rule whose output tokens are a **subset of the raw-issue tokens** (extraction, never generation) that STRIPS code-identifier tokens (paths, dotted/CamelCase symbols, stack-trace frames, quoted error strings) so the query is natural-language-shaped and structurally incapable of injecting gold vocabulary; every stripped token is recorded per case for audit; pre-registered `MECHANICAL_RULE_HASH`. `llm_distill_guarded` is a LABELED, non-deciding SENSITIVITY arm (injected `Callable`) gated by a post-hoc token-subset **HARD REJECT** (`DistillRejected`, subset-violating output rejected not passed through); pre-registered `LLM_PROMPT_HASH`. The smart arm never decides — it only disambiguates the one case the blind arm cannot (a flat mechanical delta: real null *or* the rule under-distilled). (spec 0023)
