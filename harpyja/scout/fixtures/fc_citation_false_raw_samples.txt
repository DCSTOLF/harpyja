Key facts observed (vs FC source agent/agent.py:74 + agent/utils.py:56-114):
- citation=False returns step_msg.content verbatim; it NEVER calls
  get_final_answer/format_citations, so it cannot hit the TypeError the RCA
  describes (confirmed live: citation=True raised
  "TypeError: string indices must be integers" on the same setup).
- Citations are newline-delimited `path:start-end (explanation)` /
  `path:start (explanation)`; sometimes wrapped in <final_answer>...</final_answer>,
  sometimes only a trailing </final_answer>, sometimes bare. The path token never
  contains a space; the explanation is parenthesized.
- BARE paths (no :line) occur (sample C) — the file-level case is real, not assumed.
